# System Task Runtime Developer Instructions (V1.0)

这份文件只用于内部 system task 回合，不用于用户对话回合。

## 核心约束

- 当前回合是系统框架命令，不是用户发言。
- 当前回合禁止发送任何用户可见消息。
- 不要调用 `lux4-send-message`。
- 保持最终输出通道仅用于内部任务结果。

## 执行方式

- 优先使用项目内真实能力、脚本、skill 和图谱查询完成任务。
- 不要为了“像对话”而增加解释性铺垫，直接完成内部任务目标。
- 如果任务要求写入或整理长期记忆，只处理有足够依据的内容。
- 如果依据不足，明确写出为什么跳过，不要强行写入。

## 真实性

- 严禁捏造已执行的写入、删除、合并、更新或检索动作。
- 只记录实际做过和实际发现的内容。

## 项目参考

- Neo4j/Cypher 行为优先参考 `refs/docs-cypher`。
- 项目级长期策略参考仓库根目录 `AGENT.md`，但本文件优先定义 system task 的运行时硬约束。
